## 2. ✅ 확정: 페르소나 6인

### 2.1 페르소나 정의

| # | 레이블 | 핵심 렌즈 | 대표 질문 스타일 |
|---|---|---|---|
| 1 | **토스 스타일 PO** | 숫자 · 한 문장 스펙 · 지금 증명 | "이게 성공하면 어떤 숫자가 움직여?" |
| 2 | **당근 스타일 시니어 PD** | 생활 맥락 · 로컬 · 비전문가 친화 | "우리 엄마도 이걸 쓸 수 있어?" |
| 3 | **카카오 스타일 디자인 센터장** | 시스템 · 3년 뒤에도 유효한가 | "내년 다른 프로덕트에서도 이 규칙 쓸 수 있어?" |
| 4 | **네이버 스타일 시니어 PD** | 정보 위계 · 마이크로카피 · 밀도 | "첫 3초 안에 뭘 읽게 되지?" |
| 5 | **라인 스타일 글로벌 PM** | 보편성 · 로컬라이제이션 · 문화 중립 | "일본/대만 사용자가 이걸 보면?" |
| 6 | **우아한형제들 스타일 CBO** | 감성 · 재미 · 브랜드 톤 | "이 화면에서 기분 좋은 순간은 어디야?" |

### 2.2 디자인 원칙 충돌 풀이

원래 PRD 원칙: **"회사 로고는 쓰지 않는다. 페르소나는 관점이지 브랜드가 아니다."**
실명 회사 사용과 충돌 → **풀이 2 채택**: `"○○ 스타일"` 접미사 + 디스클레이머.

**디스클레이머 문구 (UI에 명시 필요)**:
> 각 회사의 공개된 디자인 철학과 블로그/컨퍼런스 발언을 기반으로 재구성한 가상의 페르소나입니다. 실제 해당 회사 또는 직원의 의견을 대변하지 않습니다.

**금지**: 회사 로고/색/폰트 차용. 텍스트 레이블만.

### 2.3 페르소나 객체 스키마 (다음 세션에서 코드화 예정)

```ts
// 가안 — 미확정
type Persona = {
  id: string;            // 'toss-po', 'daangn-spd' 등

  firstLens: string;     // 핵심 렌즈
  nonNegotiables: string[]; // 절대 양보 안 하는 것
  tradeoffs: string[];   // 양보 가능한 것
  questionStyle: string; // 대표 질문 스타일
  systemPrompt: string;  // Claude API용 system prompt
};
```

### 3.2 강한 충돌 쌍 5개 — 충돌 테마

| 쌍 | 충돌 테마 | 프로덕트 프레이밍 |
|---|---|---|
| **1 ↔ 6** | 숫자 vs 감성 | "전환율 vs 기분 좋음, 어느 쪽?" |
| **1 ↔ 3** | 지금 증명 vs 3년 후 시스템 | "이번 분기 지표 vs 3년짜리 일관성" |
| **2 ↔ 5** | 로컬 맥락 vs 글로벌 보편성 | "동네 맥락 vs 해외에서도 통할까" |
| **4 ↔ 6** | 정보 밀도 vs 감성 여백 | "많이 보여줄까 vs 숨통 줄까" |
| **5 ↔ 6** | 문화 중립 vs 한국적 감성 | "어느 나라 사용자도 OK vs 한국인만 아는 그 느낌" |

### 3.3 매트릭스 설계 의도 (메모)

- **6번 CBO가 3개 강한 충돌의 한쪽 담당**: 유일한 "감성 진영"이라 의도된 비대칭. 나머지 5명은 이성 축에서 각자 다른 방향으로 충돌.
- **중간 충돌(🟡)도 카드화 가능**: "둘 다 맞는데 우선순위 다름" 케이스로 보여주면 PD가 스스로 절충.
- **⚪는 카드 안 만듦**: 축이 달라서 비교가 무의미.
